Scientific Article Summarization

Objective

Need to come up with the most effective, appropriate and concise summary of a scientific article, given a dataset of scientific articles.

Extractive

Several techniques were used for finding the similarity between sentences like -
Common Words
Cosine Similarity
Word Embeddings

Model

Graph Based We create a matrix to document the similarity score between any two sentences. The idea is to choose sentences which are similar to many other sentences

Scoring

Multiple heuristics common words, cosine similarity and page rank. Various hyperparameters were used to weigh the importance of each of the scoring algorithms while creating the summaries

Abstractive

Involves building a summary of the text in the way that a human would build one. We pick ideas from several paragraphs or sentences, build readable sentences and present them in a concise form.

Architecture

Used an Encoder Decoder Model with Attention Mechanism

Our architecture: 3 hidden layers per encoder/decoder, of 1024 hidden units per cell. We also used a batch size of 128 and stored checkpoints every 50 steps. Output dropout rate of 0.3 between layers.

  • Created multiple models(bucketing), with same weights to decrease the amount of padding required to generate an output. This is due to the fact that our abstracts and summaries are of greatly varying lengths.
  • The buckets are used to match with the input-output pairs. Based on the length of the input summary and the output abstract, we match it to the nearest bucket
  • Each bucket corresponds to a full encoder-decoder LSTM model, however the weights used are common across them.
  • Attention used to give additional weight to the output hidden states and current state. It allows the decoder to pay attention to different parts of the source sequence at different decoding steps.
